# Sprint 71

## 目标

- 修复 `/#im` 消息页里飞书用户名 / 群名不够明显、经常看起来只有 `ou_...` / `oc_...` 的问题
- 为飞书 IM 事件补齐用户名的落盘链路，减少对额外查询权限的依赖
- 同步更新 `docs/sprint/README.md`、`docs/prd/README.md`、`docs/tech/README.md`、`docs/manual/README.md`

- [x] IM 消息页改成直接显示“名字 + ID”
- [x] 飞书消息解析补齐可内联获取的用户名并落盘

## 设计说明

- `/#im` 不只显示原始 ID，而是优先显示 `用户名（ou_...）` / `群名（oc_...）`
- 若飞书权限不足或接口未返回名称，页面明确展示“未解析用户名 / 群名”，避免误以为功能没生效
- 能从飞书事件体直接拿到的用户名优先落盘，后续页面读取时不必再依赖补查接口
